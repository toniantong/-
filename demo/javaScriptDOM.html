深入理解 Document 與 Element 的操作

1. 什麼是 DOM？

DOM (Document Object Model，文件物件模型) 是一個程式介面。當瀏覽器載入 HTML 網頁時，它會將所有的 HTML 標籤轉換成一個個的「物件 (Object)」，並形成一個樹狀結構。

你可以把 DOM 想像成一棵倒過來的樹，最上面是 document 物件，底下是各種 HTML 標籤，每個標籤都是一個 element 物件。

核心概念：

  • document：代表整個網頁的入口點。你要找任何東西，都要從它開始。
  • element：代表網頁中的單一節點 (標籤)，例如一個 <div>、一個按鈕或一段文字。

2. Document：網頁的統治者

document 物件提供了許多方法，讓我們可以在網頁中尋找現有的元素，或是建立新的元素。

A. 尋找元素 (Selecting Elements)

最常用的兩個方法是 getElementById 和 querySelector。

// 透過 ID 尋找單一元素
const myTitle = document.getElementById('title-id');

// 透過 CSS 選擇器尋找 (尋找第一個符合的)
const myBox = document.querySelector('.box-class');

// 透過 CSS 選擇器尋找所有符合的
const allBoxes = document.querySelectorAll('.box-class');

B. 建立新元素 (Creating Elements)

使用 createElement 可以在記憶體中做出一個新的 HTML 標籤，然後用 appendChild 把它放進網頁中。

3. Element：操作單一標籤

一旦你透過 document 找到了某個 element（元素），你就可以開始改變它的長相、內容或屬性。

常用屬性與方法

  • element.textContent：修改或讀取元素的純文字內容。
  • element.innerHTML：修改或讀取元素的 HTML 結構。
  • element.style：直接修改 CSS 樣式（行內樣式）。
  • element.classList：新增、移除或切換 CSS 類別 (Class)。

4. 事件 (Events)：讓元素活起來

前面示範中一直在使用的 addEventListener 就是用來監聽使用者在 element 上做的動作（例如點擊、滑鼠移動、鍵盤輸入等）。

常用事件類型：

  • click：當使用者點擊元素時觸發
  • input：當輸入框的值改變時立刻觸發
  • change：當輸入框失焦且值有改變時觸發
  • mouseover / mouseout：滑鼠進入/離開時觸發
  • DOMContentLoaded：HTML 載入完成後觸發

5. 完整範例程式碼

// 等待整個 HTML 載入完成後才執行 JS
document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM 已經完全載入！');
    
    // 現在可以安全地操作 DOM 元素了
});

📚 重要提示

  • 永遠在 </body> 前引入 JavaScript，或使用 DOMContentLoaded 事件等待 DOM 準備好。
  • 使用 querySelector 比 getElementById 更靈活，但稍微慢一點。
  • 修改 innerHTML 時要小心 XSS 攻擊。如果內容來自使用者，使用 textContent 會更安全。
  • 事件委派 (Event Delegation) 可以大幅提升效能。
